D3 Workshop

Fundamentals of D3 / the Workshop

A small push to help you cross the initial barrier of enter the magic land of D3.

What to expect from this Workshop

This is a serie of basic exercises to serve as an introduction to the D3 library. We are gonna focus in the basics of D3 to try to solve the initial possible difficulties. We will do some exercises that will cover enough to allow you to continue flying solo.

Pre-requisites

Basics understanding of HTML and JS will be enough

How to execute the exercises

The demos of the exercises are listed below.

The JS code is using EMACSScript6.

syntax so you will need to open them using a browser that supports this syntax. I am using Chrome Version 72.0.3626.96.

If you are running the exercises in your local you will need to run them using a web server. The reason is that we are using AJAX to load the CSV data files and AJAX doesn't support to load files directly from your hard drive, it has to be through HTTP.

Disclaimer

Time pressure: I will do my best to complete the workshop in 1 hour. This is why we can not do a real full demo of each exercise. Some times I will try to do some real time code but if I see we are finding difficulties I will jump to the pre-written version.

I am not in any way a D3 expert. All the knowledge I am sharing here has been acquired in the last weeks mostly based on try and error and some video tutorials and books. So my code can have bugs. I also tried to follow the code style standards of the D3 community but some mistakes may have happened.

The examples are not very aesthetically appealing. I did have not time to prepare more WOW effect examples. It is a bit sad because I think the big potential of D3 is precisely its capacities to customize our visualizations to the infinite. If you want to see examples of the D3 aesthetic capabilities I recommend go to places like:

  • Workers’ Comp Benefits
  • Explained Visually
  • and my favourite Data Sketches

Where to go next

Try yourself. Find a nice data set and imagine how it should be represented then try to do it

Interactive Data Visualization for the Web, 2nd Edition, but only after you have tried by yourself a bit. Approaching a book directly with not any preliminary contact with D3 has not the same positive effect. Confronting the problems and developing the questions in your head is essential step to get the best of this book (or any), because it is then when you are fully receptive to the "aha!" moments that makes the knowledge to create roots in your brain. This book is fantastic, written in a very friendly language and using familiar code patterns that you will be happy to embrace when you surf the code examples of the community.

The Big Book of Dashboards, it is not in any way a book specific for D3 but it worked as a trigger to me to understand that sometimes numbers require to talk in a specific way that normal vanilla Charts can not provide.
